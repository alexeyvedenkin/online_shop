## Проект, имитирующий работу интернет-магазина
### На данный момент в проекте реализован следующий функционал:
#### - определены классы Product и Category
#### - определен вспомогательный класс ProductIterator
##### - для класса Product атрибут price определен как приватный
##### - в классе Product реализовано интерактивное подтверждение уменьшения цены
##### - в классе Product реализовано сопоставление нового экземпляра с уже имеющимися
##### - в классе Product реализован подсчет общей стоимости товаров на складе
#### - для класса Category добавлены два атрибута класса: 
##### - количество категорий (category_count) и количество товаров (product_count)
##### - для класса Category атрибут products определен как приватный
##### - в классе Category реализован подсчет общего количества товаров в категории
#### - реализованы тесты, которые проверяют:
#####   - корректность инициализации объектов класса Category,
#####   - корректность инициализации объектов класса Product,
#####   - корректность изменения цены в классе Product,
#####   - подсчет количества продуктов,
#####   - подсчет количества категорий.
#### Определены подклассы Smartphone и LawnGrass на базе класса Product
#### Для подклассов Smartphone и LawnGrass реализованы тесты, которые проверяют:
#####   - корректность инициализации объектов подклассов
#####   - корректность работы установленного метода __add__ в подклассах
#### Реализован класс NonPositiveProductQuantity, определяющий параметры самостоятельных исключений в проекте
#### Определен абстрактный класс Trading для задания параметров классов, связанных с торговлей
##### Абстрактный класс Trading определен в качестве родительского для классов Category и Order
#### Определен класс Order, определяющий формирование заказа покупателем (доп.задача)
#####   - в рамках дополнительной задачи предусмотрена возможность включения в заказ элементов, не принадлежащих классу Product (например, Transportation(грузоперевозки) или/и InstallationWorks(монтажные работы) и т.п.)
#####   - на текущий момент при инициализации экземпляра класса Order принимается продукт, количество продукта по указанию покупателя и возвращается общую стоимость заказа
#### Для подкласса Order реализованы тесты, которые проверяют:
#####   - корректность инициализации объектов подкласса
#####   - корректность работы установленного метода add_product в подклассе
#####   - корректность определения стоимости заказа
